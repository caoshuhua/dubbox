API列表>退单>批量退单查询

API功能描述

API接口：mb.returnorder.batch.get

功能：批量退单查询

授权类型：需要

应用场景：通用应用 自研应用

系统级别输入参数

名称 类型 是否必须 描述
appkey String 是 应用的app_key
token String 是 签名
timestamp String 是 时间戳，格式为yyyy-MM-dd HH:mm:ss，例如：2011-06-16 13:23:30。美邦API服务端允许客户端请求时间误差为6分钟

应用级别输入参数

名称 类型 是否必须 示例值 描述
start_date  String  否 2016-02-01 17:15:00   查询1个月内（当前数据库时间往前1个月）退单更新时间开始，格式：yyyy-MM-dd HH:mm:ss；如结束时间为空，则取起始时间到当前数据库时间的退单，如2个时间均为空就取当前数据库时间往前1个月的退单，需判断结束时间大于等于起始时间，起止时间需要在数据库当前时间往前1个月之间  
end_date  String  否 2016-02-15 17:15:00   查询1个月内（当前数据库时间往前1个月）退单更新时间开始，格式：yyyy-MM-dd HH:mm:ss；如结束时间为空，则取起始时间到当前数据库时间的退单，如2个时间均为空就取当前数据库时间往前1个月的退单，需判断结束时间大于等于起始时间，起止时间需要在数据库当前时间往前1个月之间  
returnorder_state  String  否 10   退货单状态(0,申请退货 : 1,退货中 : 10,退货完成 : 4,撤销) 
page_no  String  否 1   页码。取值范围：大于零的整数；默认值：1 
page_size  String  否 30   每页条数。取值范围：大于零的整数；默认值：30；最大值:100 

返回结果

名称 类型 示例值 描述
resp_code String 0 0为正常，其它值参见错误代码一览表
resp_info String 成功   成功 或者 失败信息  
total_results  int  1234   搜索到的退单信息总数 
return_orders  List<ReturnOrder>  1234   搜索到的退单信息列表
